The `Group.js` file defines a **Mongoose model**, which acts as a blueprint for how data is stored in MongoDB. It first imports Mongoose, then creates a schema (`GroupSchema`) that specifies the fields (like `groupname`, `email`, `mobile`, etc.), their data types, and validation rules such as required fields, lowercase formatting, and regex patterns to control allowed characters. For example, `groupname` and `email` must be strings, required, lowercase, and match specific formats. After defining the schema, it is converted into a **model** using `mongoose.model`, which allows you to create, validate, and save data (documents) into the database. This model is exported so it can be used in the main application to interact with the database consistently and safely.
